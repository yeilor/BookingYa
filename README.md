# BookingYa

Proyecto BookingYa – Gestión de Reservas (TDD, BDD y ATDD)

---

# DESCRIPCIÓN

Este proyecto tiene como objetivo implementar y demostrar diferentes enfoques de pruebas de software para garantizar la calidad en la gestión de reservas dentro de una plataforma digital.

Se desarrolló en tres fases principales:

---

## FASE 1 – TDD (Test Driven Development)

En esta fase se implementaron pruebas unitarias en Java siguiendo el enfoque TDD, donde primero se crean las pruebas y luego se desarrolla la lógica.

### Funcionalidades cubiertas:

* Creación de una reserva
* Consulta de una reserva por usuario
* Obtención de una reserva por ID
* Actualización de una reserva existente
* Eliminación de una reserva

---

## FASE 2 – BDD (Behavior Driven Development)

En esta fase se implementaron pruebas de comportamiento utilizando Serenity y Cucumber, escribiendo escenarios en lenguaje Gherkin.

### Características:

* Definición de escenarios en lenguaje natural (Given, When, Then)
* Uso de Step Definitions para ejecutar la lógica
* Validación del comportamiento del sistema desde la perspectiva del usuario

---

## FASE 3 – ATDD (Acceptance Test Driven Development)

En esta fase se desarrollaron pruebas de aceptación utilizando Playwright con TypeScript.

### Características:

* Simulación del comportamiento del sistema
* Validación de criterios de aceptación
* Escenarios implementados:

    * Crear reserva
    * Consultar reserva por ID
    * Actualizar reserva
    * Eliminar reserva

---

# OBJETIVO DEL PROYECTO

Integrar tres niveles de pruebas:

* **TDD** → Validación de la lógica interna
* **BDD** → Validación del comportamiento del sistema
* **ATDD** → Validación de los criterios de aceptación del usuario

Esto permite una cobertura completa desde el desarrollo hasta la validación final del sistema.
